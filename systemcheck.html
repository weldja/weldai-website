Part 2 — Items Requiring IT / Administrator Confirmation

The script cannot check these automatically without Administrator rights. Please ask your IT contact to confirm the following, and include their answers in your email.

🔒 Please ask your IT team to confirm:

WSL2 enabled
Windows Subsystem for Linux 2 must be enabled. To check: open PowerShell as Administrator and run wsl --status. Should show "Default Version: 2".
Virtual Machine Platform
Required for Docker. To check: open PowerShell as Administrator and run Get-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform. Should show "State: Enabled".
Docker service startup type
Docker must start automatically after a reboot. To check and fix: open PowerShell as Administrator and run Set-Service -Name "com.docker.service" -StartupType Automatic.
Firewall rules (ports 80 and 443)
Required only if multiple staff will access Weld AI from their own PCs. IT needs to open inbound TCP ports 80 and 443 on the server's Windows Firewall. Single-user installs can skip this.
Static IP address
The server should have a fixed IP address so staff can always find it. IT can assign a static IP or create a DHCP reservation. The current IP will appear in the script output — please confirm whether it is static or DHCP.

Part 3 — Installation Information

Please provide the following information in your email. This allows us to prepare your installation in advance.

The name shown in the app header — e.g. "Smith & Partners Accountants"

Approximate number — helps us advise on hardware and access configuration

The full path to the folder containing your company documents — e.g. C:\CompanyDocs or D:\Policies.

Important — documents must be stored locally: Documents must be on a local drive on the server running Weld AI. Network shares (e.g. \\server\documents) and mapped network drives (e.g. U:\docs) are not supported. If your documents are on a file server, either copy them to a local folder or install Weld AI directly on the file server.

Important — OneDrive / SharePoint users: If your documents are in a OneDrive or SharePoint synced folder, you must ensure the folder is set to "Always keep on this device" before indexing. Cloud-only files (shown with a ☁️ icon in File Explorer) are not stored locally and cannot be indexed.

To fix: right-click your documents folder in File Explorer → Always keep on this device. All files will show a ✅ green tick when ready.

Rough estimate — e.g. "around 50 PDFs" or "500+ files across several folders"
